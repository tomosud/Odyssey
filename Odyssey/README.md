Homer『オデュッセイア』を、Samuel Butler の散文英訳(Project Gutenberg #1727)を
底本として、現代文の日本語に訳し、Obsidian のデータベースとして整備するプロジェクト。
**このファイルは翻訳・整備の記述ルールを定める。セッションをリセットして再開する
場合も、まずこのファイルを読んでから作業を続けること。**

## 2. 文体ルール(最重要)

### 2-1. 地の文(ナレーション)は **常体(だ・である体)**

物語の地の文は、必ず常体で書く。過去の叙述は「〜た」で結ぶ。
**敬体(です・ます体)にしてはならない。**

- ✅ 良い例(第1・5・6歌):
  - 「さて、暁の女神がティトノスの傍らの寝床から起き出し、神々にも人間にも光を届けはじめたころ、神々は評議の席に集まった。」
  - 「こうしてオデュッセウスは、疲れと眠気に呑まれて眠っていた。その間にアテナは、パイアケス人の国と町へ飛んだ。」
  - 「さて、戦いでも難破でも死をまぬがれた者たちは、みな無事に故郷へ帰り着いていた。ただ一人、オデュッセウスだけが帰れずにいた。」
- ❌ 悪い例(第13〜15歌で混入してしまった敬体。**要修正**):
  - 「さて、アテナは、ラケダイモンの美しい町へ**向かいました**。」→ 正:「向かった。」
  - 「女神は……眠っている彼とペイシストラトスを**見つけました**。」→ 正:「見つけた。」

主な変換:
`〜ました → 〜た` / `〜です → 〜だ・である` / `〜でした → 〜だった` /
`〜ません → 〜ない` / `〜ください(地の文の説明)→ 文脈に応じ常体化` /
`〜います → 〜いる` / `〜ありました → 〜あった`

### 2-2. 会話文(「」内)は **話者と相手の関係で敬体/常体を使い分ける**

会話の丁寧さは、話し手と聞き手の社会的な上下・親疎で決める。地の文の常体とは
別に扱う。

- **神が身内・格下に**(例:ゼウス→他の神々、自分の子):ぞんざいな常体
  - 「おいおい、わが子よ、何を言っているのだ」(ゼウス)
- **目下が目上に/客を丁重にもてなすとき**:敬体
  - 「尊いお方、いつでも歓迎のお客さま……おもてなしをさせてください」(カリュプソ→ヘルメス)
- **オデュッセウスが他人に長い身の上を語るとき**:敬体を基調にしてよい
  (第9〜12歌のパイアケス人への語り、第14歌のエウマイオスへの作り話など)
- 対等・親密な間柄、命令・叱責などは常体で緩急をつける

要は「**地の文=常体で固定、会話文=場面に応じて敬体/常体**」。

### 2-3. 全体の調子

- 逐語訳ではなく、Butler の散文訳の調子を活かした読みやすい現代文にする。
- 飾り・古語・文語調は入れない(プレーンな現代文)。
- 原文の段落はすべて訳す(省略しない)。脚注参照の数字も落とさない。

## 3. 神名・人名(ローマ名 → ギリシャ名)

Butler 訳は神名・人名に**ローマ名**を使う。和訳では一般に馴染みのある**ギリシャ名**に
置き換える。各歌末尾の「訳注」に、その歌で使った対応を明記する。

| Butler(ローマ名) | 和訳(ギリシャ名) |
|---|---|
| Ulysses | オデュッセウス |
| Minerva | アテナ |
| Jove | ゼウス |
| Neptune | ポセイドン |
| Juno | ヘラ |
| Diana | アルテミス |
| Apollo | アポロン(そのまま) |
| Venus | アフロディテ |
| Mars | アレス |

## 4. 脚注の扱い

- 原文の脚注参照は、本文中では語末に付いた素の数字(Butler の連番、全編通し 1〜187)。
- 和訳本文では該当箇所に `[n]` の形でマーカーを残す。
- 対応本文は `01_原文/99_脚注(Footnotes).md`。必要に応じて各歌末尾の「訳注」でも補足する。

## 5. 和訳ファイルのテンプレート(`02_和訳/NN_第N歌_和訳.md`)

```
---
title: "第N歌 和訳"
book: N
roman: (ローマ数字)
heading_ja: "(和訳見出し)"
status: 下訳(第1稿)        # 未着手 → 下訳(第1稿) → …
---

# 第N歌 (和訳見出し)

原文 → [[NN_Book_XX]] / 脚注 → [[99_脚注(Footnotes)]] / 事典 → [[人物索引]]・[[地名・民族索引]]

> [!info] この訳について
> Butler英訳(散文)からの重訳。飾りを加えないプレーンな現代文。神名・人名はギリシャ名に置き換えている(末尾の訳注参照)。

> [!note] 物語背景
> (その歌の位置づけ・前後関係を数行で)

> [!tip] この歌の解説
> - (主題・技法・読みどころを箇条書きで)

---

(全訳。地の文は常体、会話文は場面に応じて敬体/常体。脚注マーカー [n] を残す)

---

## 訳注

- **神名・人名**:(この歌で置き換えたローマ名→ギリシャ名)
- **[n] …**:(必要な語注)
```

各歌の頭に置く**ノート**は 2 種類:
- `> [!note] 物語背景` — その歌が全体のどこに位置し、何が起きるか。
- `> [!tip] この歌の解説` — 主題・ホメロス的技法(頓呼法、定型句、クレタ人の嘘話 等)・読みどころ。

## 6. 事典(`04_事典/`)の維持ルール

翻訳の進行に合わせ、**その歌で新たに登場・言及された人物/地名/民族を必ず追記する。**

- ファイル:`人物索引.md`(人物・神)、`地名・民族索引.md`(`## 地名` と `## 民族・集団` の 2 節)
- 表の列:`名前 | 原文表記 | 説明 | 登場`
  - 「原文表記」= Butler の表記(ローマ神名など)
  - 「登場」= 登場・言及のあった歌番号。言及のみは `N(言及)` と付す。連続は `1–4` の形。
- **五十音順**に並べる(実装上は名前カラムの Unicode 昇順ソートでほぼ一致)。
- **同名別人**は必ず区別し、説明末に「※〜とは別人」と注記する。
  (例:大アイアス/小アイアス、ライストリュゴネス王アンティパテス/メランプスの子アンティパテス 等)
- frontmatter の `updated_through:` を、追記した最新の歌に更新する(例:`第15歌`)。
- 既存項目も、その歌に登場したら「登場」欄に歌番号を追記する。

追記は Python スクリプトで一括処理してよい(既存行の「登場」末尾書き換え + 新規行追加 →
名前カラムでソート)。処理後は `WARN`(想定した旧値と不一致)が出ていないか必ず確認する。

## 7. マスター索引(`00_オデュッセイア索引.md`)の更新

各歌を仕上げるたびに次の 3 点を更新する:

1. frontmatter の `status:`(例:`和訳進行中(第1〜15歌 下訳)`)
2. 「進行状況」の和訳チェックリスト(例:`和訳(段階的) — 15 / 24(第1〜15歌: 下訳)`)
3. 見出し対応表の当該歌の「和訳」列リンクを `未着手` → `下訳` に変更

  ```
  git push origin main
  git checkout <branch>
  ```

### 文体チェックの方法(参考)

- 地の文(「」の外)に敬体が残っていないかは、引用の内外を判定して走査するスクリプトで確認する。
  変換は日本語の音便(五段動詞の連用形→た/だ)を正しく扱う必要があり、単純な文字列置換
  (例:`ました→た`)は「積みた/行きた」等の誤りを生むので不可。一段/五段が表層で
  曖昧な語(起きる/落ちる/居る等)は個別に確認する。
